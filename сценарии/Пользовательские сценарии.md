### 1. Врач
**Цель:** Провести  прием пациента и оформить необходимую медицинскую документацию.

| Шаг | Действие пользователя                                                                                                                           | Реакция системы                                                                                                                                                                                      | Критерий успеха                                                     |
| --- | ----------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| 1   | Доктор входит в систему, используя двухфакторную аутентификацию.                                                                                | Система отображает "Рабочее место врача" с списком пациентов на сегодня. Рядом с именем Пациента указан статус "Ожидает".                                                                            | Врач видит свой план на день.                                       |
| 2   | Нажимает на строку с именем пациента.                                                                                                           | Система открывает карточку приема и Электронную Медицинскую Карту (ЭМК) пациента.                                                                                                                    |                                                                     |
| 3   | В разделе "Жалобы и анамнез" начинает вводить текст. Система предлагает шаблонные фразы и структурированные поля (температура, давление).       | Система сохраняет введенные данные в реальном времени. Интегрируется со справочником МКБ-10 для быстрого поиска кодов диагнозов.                                                                     | Данные вносятся быстро, с минимальным ручным вводом.                |
| 4   | На основе осмотра и анамнеза устанавливает предварительный диагноз "Острый бронхит" (J20.9). Вносит назначения: лекарства, режим, обследования. | Система проверяет назначения на потенциальные аллергии и лекарственные взаимодействия (если такие данные есть в карте).                                                                              | Повышение безопасности пациента.                                    |
| 5   | Врач принимает решение оформить Листок нетрудоспособности (больничный). Нажимает кнопку "Создать ЭМД" и выбирает соответствующую форму.         | Система открывает шаблон электронного больничного листа, автоматически подтягивая данные пациента, врача и диагноза.                                                                                 | Документ создан в требуемом формате.                                |
| 6   | Врач проверяет данные и нажимает "Подписать ЭП".                                                                                                | Система инициирует процесс подписания с использованием токена. После успешного подписания документ сохраняется в ЭМК и автоматически отправляется в ФГИС. Статус меняется на "Подписан и отправлен". | Документ приобрел юридическую силу и передан в вышестоящую систему. |
| 7   | Врач завершает прием.                                                                                                                           | Система закрывает карточку приема, обновляет статус пациента на "Прием завершен".                                                                                                                    | Прием корректно завершен                                            |

### 2. Регистратор

**Цель:** Зарегистрировать нового пациента в системе и перенести запись существующего пациента.

| Шаг | Действие пользователя                                                                                                                                          | Реакция системы                                                                                                                       | Критерий успеха                            |
| --- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------ |
| 1   | Пользователь входит в систему под своей учетной записью с правами "Регистратор".                                                                               | Система открывает интерфейс регистратора с разделами: "Регистрация пациента", "Расписание", "Поиск пациента".                         |                                            |
| 2   | В поликлиннику обращается новый пациент. Регистратор нажимает "Создать новую карту".                                                                           | Система открывает анкету с полями: ФИО, пол, дата рождения, полис ОМС, СНИЛС, адрес, контакты.                                        |                                            |
| 3   | Регистратор вводит данные пациента.                                                                                                                            | Система в реальном времени проверяет полис ОМС и СНИЛС на уникальность, предотвращая создание дубликатов карт.                        | Создан профиль пациента.                   |
| 4   | Сохраняет анкету.                                                                                                                                              | Система создает новую ЭМК и присваивает пациенту уникальный идентификатор (номер карты).                                              |                                            |
